Guigen - wizards di modellazione struttura UI

wizard classe di attivazione condizioni di attivazione descrizione
SimplePageStructure: crea struttura pagina (con scelta template) ContentPanel il ContentPanel non deve già contenere nessun pannello Crea la struttura di primo livello della schermata. Permette di scegliere:
  • quanti quadranti attivare (up/down/left/right/center)
  • il titolo della schermata
  • l'etichetta del pannello centrale
  • se visualizzare il menu, e in caso affermativo se visualizzarlo in alto (menu orizzontale) o a sinistra (menu verticale)
  • se vusualizzare il pannello di infornazioni utente
FormFields4RecordAppdata: crea struttura di campi a partire da application data di tipo struttura FormPanel il ContentPanel che contiene il FormPanel selezionato deve referenziare preventivamente almeno un ApplicationData di tipo ComplexType Crea una struttura di campi in base alla struttura dati a cui fa riferimento un ApplicationData e effettua i binding tra i Widget e i Field. Il wizard presenta all'utente le seguenti scelte:
  • quale tra gli ApplicationData referenziati dal ContentPanel contenitore deve essere utilizzato come fonte dati
  • quale deve essere l'etichetta del WidgetsPanel che conterrà i Widget
  • quali tra i Field del ComplexType visualizzare
Il pannello creato avrà un layout verticale e i Widget saranno creati in base al tipo del campo (es. Data -> Calendar, Stringa o numerico -> TextField, booleano -> CheckBox).
Table4CollectionAppdata: crea una Table a partire da un ApplicationData di tipo array di ComplexType WidgetsPanel il ContentPanel che contiene il FormPanel selezionato deve referenziare preventivamente almeno un ApplicationData di tipo TypedArray di ComplexType Crea una Table per visualizzare il contenuto dell'ApplicationData. Il wizard presenta all'utente le seguenti scelte:
  • quale tra gli ApplicationData referenziati dal ContentPanel contenitore deve essere utilizzato come fonte dati (collection-binding)
  • quale deve essere il name della Table
  • quali tra i Field del ComplexType contenuto nella collection visualizzare
  • il tipo di selezione che sarà possibile effettuare sulla tabella (nessun selezione | selezione singola | selezione multipla)
  • in caso di selezionabilità:
    • il campo che deve essere utilizzato come identificativo del record
    • l'ApplicationData che dovrà contenere l'identificativo della riga selezionata (value-binding)
Il pannello creato avrà un layout verticale e i Widget saranno creati in base al tipo del campo (es. Data -> Calendar, Stringa o numerico -> TextField, booleano -> CheckBox).
