Reference guigen :: UISecurityConstraint securityConstraints

E' possibile specificare uno o più security constraints sul widget, che permettono di rendere condizionata la visualizzazione o la abilitazione del widget a fronte delle condizioni descritte nel constraint. I constraint sono additivi (nel senso booleano) per ciascun constrained behavior (comportamento condizionato = {visibilità / abilitazione}): se almeno un constraint è verificato il comportamento condizionato sarà abilitato (visibilità / editabilità). Se nessun constraint sarà verificato, la visualizzazioen/abilitazione sarà condizionata solo allo stato di default / comandato, ovvero al valore delle proprietà defaultVisible o defaultEnabled se non è stato in precedenza utilizzato nessun comando di tipo ONOFFCommand o VisibilityCommand; se tali comandi sono stati utilizzati allora "vince" l'impostazione derivante da questi comandi. Se, infine, un constraint è verificato, comunque un eventuale disabilitazione/invisibilità comandata (tramite ONOFFCommand o VisibilityCommand) vince.
