Identificativo e riferimenti di dettaglio

La documentazione metamodeldoc di dettaglio del widget ResetButton è disponibile qui.

Descrizione

E' il classico pulsante di reset utilizzabile per ripulire la forma utente, riportandola allo stato iniziale. Il ResetButton può essere utilizzato all'interno di:
  • WidgetsPanel, con layout Verticale, Orizzontale, Griglia, nel caso sia necessario associare il pulsante ad altri widget (es. pulsanti di lookup)
  • CommandPanel, con layout Verticale, Orizzontale, Griglia, nel caso sia necessario realizzare una "pulsantiera" costituita da soli Button
Prevede una sola modalità di funzionamento: standard.

modalità standard

Il funzionamento di base del ResetButton prevede:
  • la possibilità di essere disabilitato a comando
  • la possibilità di essere reso invisibile a comando
  • la possibilità di essere disabilitato/invisibile a fronte di regole associate al profilo dell'utente collegato
  • la possibilità di associare al pulsante uno specificatore di funzione, che può essere utilizzato per connotare la funzione che eseguirà tale pulsante, con l'effetto, ad esempio di permettere l'associazione di uno stile specifico o di un'icona rappresentativa della funzone stessa. L'elenco delle funzioni specificabili è il seguente:
    • ADD_ITEM: aggiunta di un elemento ad un insieme di dati/tabella
    • BACK: ritorno ad una schermata precedente
    • CANCEL: risposta negativa ad una richiesta di conferma di esecuzione di comando
    • CONFIRM: risposta positiva ad una richiesta di conferma di esecuzione di comando
    • DELETE_ITEM: eliminazione di un elemento da un insieme di dati/tabella
    • DETAIL: visualizzazione del dettaglio di una entità
    • EDIT_ITEM: inizio delle attività di modifica di un'entità
    • FIRST_ITEM: posizionamento sul primo elemento di un elenco di dati
    • FORWARD: prosecuzione al prossimo passo di una serie di passi guidati
    • LAST_ITEM: posizionamento sull'ultimo elemento di un elenco di dati
    • LOAD: caricamento di un record
    • NEXT_ITEM: posizionamento sul prossimo elemento di un elenco di dati
    • PREVIOUS_ITEM: posizionamento sull'elemento precedente a quello corrente all'interno di un elenco di dati
    • SAVE: salvataggio dei dati immessi
    • SEARCH: effettuazione della ricerca
    • SEND_MESSAGE: invio di un messaggio (es. email)
    • SHOW_HELP: visualizzazione di una pagina di aiuto
    • SHOW_REPORT: visualizzazione di un report
    • UNSPECIFIED: funzione generica

Eventi associati al widget

evento "event-clicked"

Il pulsante può ricevere un evento di pressione da parte dell'utente che scatena il reset della schermata. Questo evento è implicito (non occorre definire un EventHandler per attivarlo). A fronte di questo evento non è perciò possibile eseguire logica di business.
